================================================================================
  [최종 상세 명세서] Quine-McCluskey 프로젝트 구현 메서드 1:1 매칭 명세서
================================================================================

본 명세서는 실제 프로젝트 소스 코드(Implicant, LogicSimplifier, PIChart, Main)에 
작성된 모든 퍼블릭(Public) 및 프라이빗(Private) 메서드들을 1:1로 매칭하여, 
각 메서드의 정확한 역할과 입출력 흐름을 학술적/실습적 관점에서 상세히 기술합니다.

--------------------------------------------------------------------------------
1. Implicant 클래스 (src/Implicant.java)
--------------------------------------------------------------------------------
각 이진 항(Minterm 및 결합된 항)의 데이터 구조를 정의하며, 결합 연산을 스스로 수행하는 객체입니다.

  1. Implicant(int numVars, int mintermValue) [Public 생성자]
     - 역할: 0차(Column 0) 임플리컨트를 생성하는 기본 생성자입니다.
     - 동작: 입력받은 10진수 민텀 번호를 `toBinaryString` 메서드를 통해 지정된 변수 자릿수(numVars)에 맞는 이진수 문자열로 인코딩하여 `term`에 저장하고, 해당 민텀 값을 `minterms` 목록에 추가합니다.

  2. Implicant(String term, List<Integer> minterms) [Public 생성자]
     - 역할: 1차 이상의 결합된 상위 임플리컨트를 생성하는 생성자입니다.
     - 동작: 결합을 통해 도출된 새로운 이진 표현 문자열(`term`)과 기존에 커버하던 오리지널 민텀들의 병합 목록(`minterms`)을 입력받습니다. 외부 리스트 수정에 영향받지 않도록 새로운 `ArrayList`를 생성하여 복사 보관합니다.

  3. toBinaryString(int value, int numVars) [Private 메서드]
     - 역할: 10진수 정수형 민텀 값을 자릿수에 맞는 2진 문자열로 반환하는 이진화 헬퍼 메서드입니다.
     - 동작: 반복적인 나머지 연산(`% 2`, `/ 2`) 루프를 통해 지정된 자릿수(numVars)만큼의 '0'과 '1'로 구성된 문자열을 생성하며, 역순 출력을 방지하기 위해 나머지를 문자열 앞 쪽에 덧붙여 나가며 패딩을 처리합니다.

  4. countOnes() [Public 메서드]
     - 역할: 현재 항이 가지는 Hamming Weight(문자 '1'의 개수)를 반환합니다.
     - 동작: 문자열 `term`을 처음부터 끝까지 루프 순회하며 대시('x')는 무시하고 오직 문자 '1'의 개수만을 카운트하여 정수형으로 반환합니다. 이 반환값은 0차 및 고차 그룹 분류의 핵심 인덱스로 활용됩니다.

  5. combine(Implicant other) [Public 메서드]
     - 역할: 자신(this)과 상대방(other) 임플리컨트를 비교하여 결합성 검증 및 실제 결합 처리를 동시에 수행합니다.
     - 동작: 
       ① 두 이진 표현의 대시('x') 자리가 완벽하게 일치하는지 검사합니다.
       ② 대시 자리를 제외한 나머지 비트 중 **정확히 단 한 자리만 다른지** 카운트합니다.
       ③ 결합 요건을 충족하면 다른 비트 자리를 대시('x')로 치환한 새로운 이진 표현을 생성하고, 두 객체가 가졌던 민텀 리스트를 합쳐 `TreeSet`을 통해 중복 제거 및 오름차순 정렬을 보장한 새 `Implicant` 객체를 생성하여 반환합니다.
       ④ 결합 요건이 충족되지 않으면 **`null`**을 반환합니다.

  6. equals(Object obj) [Public 오버라이딩]
     - 역할: 두 임플리컨트 객체의 논리적 동등성을 결정합니다.
     - 동작: 두 객체가 동일한 클래스 타입인 경우, 메모리 주소가 다르더라도 멤버 변수 `term` 문자열의 값이 완벽히 일치하면 논리적으로 동일한 객체(`true`)로 판별합니다. 이를 통해 HashSet 등에서 자동으로 중복을 제거할 수 있게 돕습니다.

  7. hashCode() [Public 오버라이딩]
     - 역할: 해시 기반 컬렉션(HashSet)의 검색 효율성을 보장하는 고유 해시코드를 반환합니다.
     - 동작: `equals` 메서드와 일관성을 이루도록 `term` 문자열의 해시코드를 그대로 반환하여, 동일한 이진 표현을 가진 두 객체가 HashSet 내부에서 동일한 해시 버킷에 매핑되도록 보장합니다.

  8. toString() [Public 오버라이딩]
     - 역할: 임플리컨트 상태를 디버깅용 포맷으로 반환합니다.
     - 동작: 이진 문자열 `term`과 해당 항이 커버하는 오리지널 민텀 목록 문자열을 포맷팅하여 반환합니다. (예: `x100 [4, 12]`)

2. LogicSimplifier 클래스 (src/LogicSimplifier.java)
--------------------------------------------------------------------------------
Tabular Method(도표법)의 그룹화 및 세대별 결합 프로세스를 총괄하여 최종 Prime Implicants를 찾아내는 클래스입니다.

  1. LogicSimplifier(int numVars) [Public 생성자]
     - 역할: Simplifier 인스턴스를 초기화합니다.
     - 동작: 입력받을 부울 변수의 개수(numVars)를 멤버 변수로 할당하여, 이진법 인코딩의 자릿수 및 Hamming Weight 그룹 분류 배열 크기의 기준을 확립합니다.

  2. findPrimeImplicants(List<Integer> minterms, List<Integer> dontCares) [Public 메서드]
     - 역할: 전체 테이블 결합 루프를 제어하여 중복이 제거된 최종 Prime Implicant(PI) 목록을 반환합니다.
     - 동작:
       ① 민텀과 돈케어를 통합하여 0차 Implicant 리스트를 생성합니다.
       ② `currentColumn`에 있는 항들을 Hamming Weight (1의 개수)에 따라 0개부터 N개까지 `List<List<Implicant>>` 그룹으로 나눕니다.
       ③ i번째 그룹과 i+1번째 인접 그룹의 항들을 전부 2중 루프로 대조하여 `a.combine(b)`를 호출합니다. 결합 성공 시 두 부모 항의 `isUsed` 필드를 `true`로 설정하고 결합된 신규 항을 `nextColumnSet`에 수집합니다.
       ④ 현재 세대(`currentColumn`)의 순회가 끝났을 때 `isUsed`가 여전히 `false`인 항들(더 이상 결합에 사용되지 않은 항)을 주임플리컨트(PI) 후보로 수집합니다. 단, 돈케어만 커버하는 무의미한 PI를 배제하기 위해 **오리지널 민텀 목록 중 최소 1개 이상을 커버하고 있는 항만 최종 수집**합니다.
       ⑤ `nextColumnSet`이 비어있지 않다면 세대를 교체하여 결합 루프를 재수행하며, 더 이상 결합이 이루어지지 않을 때 루프를 종료하고 수집된 최종 PI 리스트를 반환합니다.

3. PIChart 클래스 (src/PIChart.java)
--------------------------------------------------------------------------------
도출된 PI들과 오리지널 민텀 간의 2차원 매핑 표를 구성하고, 다양한 행렬 축소 법칙 및 
백트래킹을 동반하여 모든 민텀을 커버하는 최소 개수의 Implicant 목록을 도출하는 해결사 클래스입니다.

  1. PIChart(List<Implicant> piList, List<Integer> mtList) [Public 생성자]
     - 역할: PI Chart(주임플리컨트 도표) 행렬 구조를 자동으로 빌드하고 초기화합니다.
     - 동작: 멤버 변수 `primeImplicants` 및 `minterms`를 내부 리스트로 카피하고, `primeImplicants.size() x minterms.size()` 규격의 2차원 불리언 행렬(`chart`)을 동적 할당합니다. 행/열의 삭제 여부를 추적할 삭제 매핑 배열(`isRowDeleted`, `isColDeleted`)을 동일 자릿수로 할당한 후, 각 PI가 지닌 민텀 목록에 대상 열 민텀 번호가 포함되어 있으면 `chart[row][col] = true`로 격자점을 세팅합니다.

  2. solve() [Public 메서드]
     - 역할: EPI 선택, 행/열 지배 도표 축소, 잔여 차트 해결의 전체 알고리즘을 반복 조율하여 최종 최적 해 리스트를 반환합니다.
     - 동작: 도표 내에 상태 변화가 일어나지 않을 때까지 `selectEPIs` 루프와 `reduceChartByDominance` 루프를 반복 호출하여 1차 및 2차로 차트를 극소 수준으로 축소합니다. 루프 탈출 후 아직 완벽히 커버되지 않고 남은 민텀(열)이 존재하면 `solveRemainingChartWithBacktracking`을 긴급 호출하여 잔여 순환 도표를 처리하고 최종 선택된 최적의 Implicant 목록을 병합하여 반환합니다.

  3. selectEPIs(List<Implicant> selectedPIs) [Private 메서드]
     - 역할: 미완료 열 중 단 하나의 활성 행에 의해서만 커버되는 민텀을 찾아 필수 주임플리컨트(EPI)로 선정합니다.
     - 동작: 삭제되지 않은 모든 열을 순회하며 각 열을 커버하는 활성 행(PI)의 개수를 세어, 개수가 정확히 1개인 경우 해당 행을 EPI로 선정합니다. 해당 EPI 행을 삭제 표시(`isRowDeleted = true`)하고 최종 선택 목록에 등록한 후, 해당 EPI가 커버하는 도표 내의 모든 열들을 커버 완료(`isColDeleted = true`) 처리합니다. EPI가 새롭게 발견되었다면 변경 플래그(`true`)를 반환합니다.

  4. reduceChartByDominance() [Private 메서드]
     - 역할: 행 지배(Row Dominance) 및 열 지배(Column Dominance)를 도표에 적용하여 복잡도를 줄입니다.
     - 동작:
       - 행 지배: 지배적인 행(Superset 관계를 가지는 행)이 존재할 때 피지배 행(더 좁은 범위를 커버해 선택 메리트가 떨어지는 행)을 `isRowDeleted = true`로 제거합니다.
       - 열 지배: 지배적인 열(더 다양한 PI에 의해 커버되어 해결하기 쉬운 열)이 존재할 때, 까다로운 열을 만족하면 자동으로 커버되므로 해당 지배 열을 `isColDeleted = true`로 차트에서 삭제 처리합니다. 도표가 축소된 내역이 있다면 변경 플래그(`true`)를 반환합니다.

  5. doesRowDominate(int rowA, int rowB) [Private 메서드]
     - 역할: 행 rowA가 행 rowB를 지배(Row Dominant)하는 관계인지 검증합니다.
     - 동작: 삭제되지 않은 활성 열들을 순회하며, rowB가 커버하는 모든 열에 대해 rowA 역시 완벽하게 커버하고 있는지(`Superset` 관계) 판별합니다. 만약 두 행의 커버 범위가 완전히 일치한다면, 무한 대칭 삭제(Infinite Loop)를 방지하기 위해 행 인덱스를 비교하는 타이 브레이킹 조건을 적용하여 한 행만 지배적인 것으로 판단하도록 처리합니다.

  6. doesColDominate(int colA, int colB) [Private 메서드]
     - 역할: 열 colA가 열 colB를 지배(Column Dominant)하는 관계인지 검증합니다.
     - 동작: 삭제되지 않은 활성 행들을 순회하며, colB를 만족시킬 수 있는 모든 PI들에 대해 colA 역시 동일하게 만족시킬 수 있는지 판별합니다. 두 열의 커버 구조가 완전히 일치하면 마찬가지로 인덱스를 비교하여 타이 브레이킹을 적용함으로써 한쪽 열만 지배적인 것(더 해결하기 쉬운 것)으로 결론을 내려 삭제 대상으로 지정합니다.

  7. isAllColsCovered() [Private 메서드]
     - 역할: 도표 내의 모든 오리지널 민텀 열들이 완벽하게 커버되어 삭제 완료되었는지 확인합니다.
     - 동작: `isColDeleted` 배열 전체를 루프 순회하여, 아직 커버되지 않은 채 남아있는 열(`false`)이 단 하나라도 있다면 `false`를 반환하고, 모든 열이 완료되었다면 `true`를 반환합니다.

  8. solveRemainingChartWithBacktracking() [Private 메서드]
     - 역할: 행렬 축소 기법 적용 후에도 잔여 도표가 남아있는 순환 차트(Cyclic Chart) 상태일 때 최적의 해 탐색을 개시합니다.
     - 동작: 삭제 처리되지 않고 남아있는 잔여 활성 열 인덱스 리스트(`activeCols`)와 잔여 활성 행 인덱스 리스트(`activeRows`)를 추출하여 보관합니다. 최적 조합 인덱스를 보관할 리스트를 할당한 후 `backtrack` 탐색 코어를 호출하여 가장 최소 크기의 행 인덱스 조합을 엄밀히 구해내고, 해당 인덱스들에 대응되는 `Implicant` 리스트를 최종 구성하여 반환합니다.

  9. backtrack(List<Integer> activeRows, List<Integer> activeCols, int startRowIndex, List<Integer> currentCombination, List<Integer> bestCombination) [Private 메서드]
     - 역할: 순환 차트 상태에서 재귀 루프를 돌며 모든 경우의 수를 검증하는 백트래킹 코어 연산 엔진입니다.
     - 동작:
       ① **가지치기(Pruning):** 현재 탐색 중인 조합의 크기가 이미 찾은 최적(최소) 조합의 크기보다 크거나 같다면 유망하지 않으므로 즉시 탐색을 차단하고 롤백합니다.
       ② **해 도출 검증:** 현재 조합으로 남은 모든 활성 열(`activeCols`)이 커버되는지 `coversAllCols`로 검증한 후, 커버된다면 최적 조합(`bestCombination`)을 갱신하고 재귀를 빠져나옵니다.
       ③ **분기 탐색:** 해당 행을 선택하지 않는 경우의 수 분기와, 해당 행을 선택한 경우의 수 분기(조합에 추가 후 재귀 호출, 이후 백트래킹 롤백 처리)를 각각 순차적으로 깊이 우선 순회합니다.

  10. coversAllCols(List<Integer> rowIndices, List<Integer> activeCols) [Private 메서드]
      - 역할: 백트래킹 탐색 중에 선택된 특정 행(PI) 조합이 남은 잔여 열들을 모두 만족하는지 엄밀히 판단합니다.
      - 동작: 모든 잔여 활성 열 인덱스를 하나씩 돌며, 선택된 임시 행 조합 중 단 하나라도 해당 열에 true 표시(`chart[row][col]`)를 가지고 있는지 확인합니다. 모든 활성 열에 대해 만족하면 `true`, 하나라도 빠졌다면 `false`를 반환합니다.

--------------------------------------------------------------------------------
4. Main 클래스 (src/Main.java)
--------------------------------------------------------------------------------
사용자와의 터미널 인터페이스 제공 및 전체적인 클래스간의 결합 흐름을 조율하는 오케스트레이터입니다.

  1. main(String[] args) [Public 정적 메서드]
     - 역할: 프로그램의 표준 엔트리 포인트(Entry Point)입니다.
     - 동작: `Scanner`를 통해 변수의 자릿수 N, Minterm 목록, Don't Care 목록 문자열을 유연하게 기입받은 후, `parseInputString`을 통해 정수형 목록으로 환산합니다. 순차적으로 `LogicSimplifier.findPrimeImplicants`를 호출해 1단계 PI를 도출해 터미널에 콘솔 출력하고, `PIChart` 인스턴스를 올려 `solve()`를 실행하여 2단계 최적 PI들을 얻은 뒤 `convertToSOPExpression`을 이용해 가독성 높은 SOP 표현식으로 최종 인쇄합니다.

  2. parseInputString(String input) [Private 정적 메서드]
     - 역할: 무작위 공백이나 쉼표(,)가 뒤섞여 들어오는 지저분한 콘솔 입력 문자열을 정수 목록으로 정밀하게 정제합니다.
     - 동작: 입력 문자열을 정규식 기호 `[,\\s]+`(공백 및 쉼표가 여러 번 연속되는 패턴)를 기준으로 토큰 분할(split)하고, 각 비어있지 않은 문자열 토큰을 `Integer.parseInt`로 변환하여 유효한 `List<Integer>` 리스트 구조로 반환합니다.

  3. convertToSOPExpression(List<Implicant> selected) [Private 정적 메서드]
     - 역할: 최종 간소화된 Implicant들의 이진 표현(예: `"1x01"`)을 대수학 기호 표현식(SOP, 예: `"x1x3'x4"`)으로 정규 변환합니다.
     - 동작: 각 임플리컨트의 이진 문자열 `term`의 인덱스 `i`를 돌며 `'1'`인 경우 원형 문자열 `"x" + (i + 1)`을, `'0'`인 경우 반전 문자열 `"x" + (i + 1) + "'"`을 붙여 곱항(Product Term)을 빌드합니다. 대시('x') 자리는 기호 소거 대상이므로 완벽히 무시합니다. 다수 항들을 합(+) 기호로 연동하며, 만약 모든 비트가 소거되어 항 자체가 공백이 된다면 부울 대수 법칙에 따라 상수 `"1"`로 반환합니다.
