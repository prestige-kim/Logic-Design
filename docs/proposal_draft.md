# [Phase 2] 논리설계 보너스 과제 제안서 (Proposal) 초안

이 문서는 **Handong Global University 논리설계 보너스 과제(Two-Level Logic Minimizer)**의 공식 제안서 초안입니다. 팀원들과 공유할 수 있도록 프로젝트의 목표, 문자열 기반의 직관적인 핵심 알고리즘, 클래스 설계, 개발 일정을 반영하여 대폭 수정했습니다.

---

## 1. 프로젝트 개요 및 목표 (Introduction & Objective)

본 프로젝트는 임의의 부울 함수(Boolean Function)의 Minterm과 Don't Care 조건을 입력받아, **Quine-McCluskey 방법**과 **PI Chart Reduction 알고리즘**을 구현하여 가장 간소화된 **SOP(Sum-of-Products) 식**을 도출하는 프로그램 개발을 목표로 합니다.

### 핵심 목표:
* **구현 신뢰성**: 대시(`-`) 상태 추적 및 병합 단계에서 오류가 발생하지 않도록 촘촘한 조건 비교 논리를 개발합니다.
* **직관적인 문자열(String) 기반 설계**: 복잡하고 가독성이 떨어지는 로우레벨 비트 연산 대신, 자바 프로그래밍 입문 수준에서 누구나 읽고 직관적으로 디버깅할 수 있는 문자열과 문자 단위 비교 방식을 도입합니다.
* **예외 케이스 처리**: Don't Care 분리 처리 및 순환 구조(Cyclic Table) 도달 시 감점 없이 완벽한 해를 탐색하는 로직을 수립합니다.

---

## 2. 핵심 알고리즘 및 설계 (Algorithm & System Design)

### 2.1. 문자열(String) 기반 Implicant 표현 및 병합법
변수의 개수가 최대 $N \le 10$이므로, 컴퓨터 메모리 최적화보다는 사람이 한눈에 해독하기 쉬운 **이진법 형태의 문자열**(예: `"0-10"`)을 데이터의 핵심 표현법으로 채택합니다.

* **이진 문자열 초기화**:
  * 입력된 정수 값(Minterm 번호)을 $N$ 자리의 2진수 문자열로 변환하여 저장합니다.
  * 나눗셈과 나머지 연산(`% 2`, `/ 2`)을 순차 루프로 수행하여, StringBuilder 등 어려운 API 없이 순수 자바의 String prepend(앞으로 덧붙이기) 연산만으로 구성합니다.
  * *예시 ($N=4$)*: `5` $\rightarrow$ `"0101"`

* **병합 조건 검증 (`canCombineWith`)**:
  두 Implicant $A$와 $B$는 다음 조건을 모두 만족할 때 결합할 수 있습니다.
  1. 두 문자열의 길이가 일치하며, 대시(`-`) 문자가 존재하는 위치가 완벽히 동일해야 합니다.
  2. 대시가 아닌 일반 문자 자리 중 **단 한 자리**만 서로 문자 값이 달라야 합니다. (하나는 `'0'`, 다른 하나는 `'1'`)

* **병합 수행 (`combine`)**:
  두 항의 문자들을 비교하며 서로 달랐던 단 한 자리를 대시(`'-'`) 문자로 치환한 새로운 문자열을 만들고, 두 임플리컨트의 Minterm 리스트를 결합하여 새로운 `Implicant` 객체를 생성합니다.
  * *예시*: `"0001"` 과 `"0011"` 결합 $\rightarrow$ `"00-1"`

---

### 2.2. PI Chart Reduction 및 Branching (Branch-and-Bound)

1. **EPI (Essential Prime Implicant) 추출**:
   * Don't care 조건을 제외하고, 각 Minterm 열(Column) 중에서 단 하나의 PI 행(Row)에 의해서만 커버되는 Minterm을 찾습니다.
   * 발견된 PI는 필수 주임플리컨트(EPI)로 선정되어 최종식에 무조건 합류하며, 해당 행과 커버되는 모든 열을 표에서 삭제합니다.

2. **Row & Column Domination (행/열 지배 법칙)**:
   * **행 지배**: 더 많은 Minterm 열을 포함하여 커버하는 우월한 행이 열세인 행을 지배하므로, 피지배 행을 표에서 삭제하여 표를 줄여나갑니다.
   * **열 지배**: 깐깐하게 특정 소수 PI만으로 커버되는 Minterm 열(피지배 열)이 존재할 때, 이 열을 완벽히 포함하여 쉽게 커버되는 열(지배 열)을 삭제하여 연산 대상을 좁힙니다.

3. **Branching Approach (순환 표 해결)**:
   * 단순 표 축소로 더 이상 축소가 되지 않는 Cyclic Table 상태에 이르면, **Branch-and-Bound (분기 탐색)** 방식을 적용합니다.
   * 특정 PI를 선택하는 분기와 배제하는 분기로 갈라져 각각 재귀적으로 표 축소를 시도하며, 최종적으로 곱의 항(Product Term)의 개수가 가장 적고 리터럴 문자 수가 가장 적은 최적의 SOP 조합을 선택합니다.

## 3. 소프트웨어 아키텍처 및 역할 분담 (Timeline)

### 3.1. 핵심 모듈 구성
* **`Implicant`**: 개별 항의 정보를 이진 문자열(`term`) 형태로 지니고 있는 데이터 객체. 1의 개수 세기 및 단일 결합 기능 수행.
* **`LogicSimplifier`**: 1의 개수별로 그룹화(`groupByHammingWeight`)하고 병합을 반복하여 Prime Implicant(PI) 목록을 추출하는 조율기.
* **`PIChart`**: PI와 Minterm의 관계 표를 만들고, EPI 추출, 행/열 지배 및 Branching 알고리즘을 종합 수행하는 최적화 솔버.
* **`Main`**: 콘솔 입출력을 담당하고 각 클래스 객체를 유기적으로 연결하여 작동시키는 메인 진입점.
